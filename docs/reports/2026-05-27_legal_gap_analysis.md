## 0. 本レポートの位置付け

統合 audit §5.2 (Ads / Privacy / Legal) で抽出された「広告掲載 (AdSense) / サブスク導入 (Stripe) の審査時に問題になり得る要素」について、Plan B B-6 のスコープで以下を整理する:

1. 現状の `terms` / `privacy` / `contact` 各ページの内容確認
2. 一般公開 (無料公開) 段階で追加対応が必要かどうかの判定
3. AdSense 申請段階で必要となる項目
4. サブスク (Stripe) 導入段階で必要となる項目
5. 将来の GDPR/CCPA 対応で必要となる項目 (海外展開時のみ)

**Plan B B-6 では実装はしない**。本レポートを入力として Phase 3 法務 plan を起案する。

## 2. 一般公開 (無料公開) 段階での追加対応

**結論: 追加実装なしで公開可能**。

統合 audit §5.2 で指摘された項目は、サブスク導入 / AdSense 申請のタイミングで初めて必要になるもので、現状の Plan A 完了状態 (実態) と現法務文言で一般無料公開は問題なし。

ただし以下は **継続監視項目** として残しておく:

| 項目 | 現状 | 監視点 |
|---|---|---|
| AdSense 用 ads.txt | 不存在 | AdSense 申請通過後に追加。申請前は不要 |
| AdSense 用第三者広告の Privacy 言及 | 不存在 | AdSense 通過後に privacy 文言追加 |
| Cookie consent UI (GDPR/CCPA) | 不存在 | 海外展開時に検討。日本国内のみなら短期不要 |
| 特商法に基づく表記 | 不存在 | サブスク導入時に必須 |

## 3. AdSense 申請段階で必要となる項目

| 項目 | 必要性 | 対応案 |
|---|---|---|
| 「広告 / 第三者広告事業者の利用」をプライバシーに記載 | 必須 | privacy §4 「外部サービス利用」セクションに Google AdSense を追記し、Google の広告 / Cookie ポリシーへリンク |
| 「広告 / トラッキング Cookie 利用」をプライバシーに記載 | 必須 | privacy §6 「Cookie・セッション」にも追記 |
| ads.txt | 必須 | `/public/ads.txt` に Google の publisher id を記載 (申請通過後に Google から提示される) |
| プライバシーポリシーへのアクセス導線 | 必須 | 既にフッターから到達可能 ✅ |
| 連絡先 | 必須 | `/contact` 存在 ✅ |
| Original Content (insufficient original content 警告対策) | 必須 | Plan B B-4 のランディング SSR 化で「アプリの紹介・対応ゲーム・機能説明」を SSR HTML に含める → 充足見込 |
| 暴力・成人・違法コンテンツの排除 | 必須 | カードゲーム戦績ツールなので基本問題なし |

## 4. サブスク (Stripe) 導入段階で必要となる項目

| 項目 | 必要性 | 対応案 |
|---|---|---|
| 特定商取引法に基づく表記 | 必須 | `/specified-commercial-transactions` (または `/legal/sct`) を新規追加。事業者氏名・住所 (開示請求方式可)・電話番号 (任意)・代金支払時期・引渡時期・返品特約・販売価格を記載 |
| 利用規約に有料プラン条項 | 必須 | `/terms` に「有料プラン契約 (自動更新 / 解約 / 返金不可ポリシー)」セクションを追加 |
| プライバシーポリシーへの決済情報取得記載 | 必須 | privacy §1 「取得する個人情報」に「Stripe 経由の決済情報 (カード末尾 4 桁 / 決済日時 / 金額 等)」を追記。実カード番号は Stripe 側で保持し本サービスは保持しない旨を明記 |
| Stripe を外部サービスとして記載 | 必須 | privacy §4 「外部サービス利用」に Stripe Inc. (米国) を追加 |
| Stripe Customer Portal 連携 | 必須 | アプリ内に「サブスク解約 / プラン変更 / 領収書ダウンロード」リンクを追加 |
| 自動更新の事前告知 | 必須 (特商法 / 消費者契約法) | サブスク UI で「次回更新日」「自動更新の解約方法」を明示 |
| 価格表示 (税込) | 必須 | 価格は内税表記 |
| 連絡先 | 必須 | `/contact` 存在 ✅ |
| 中途解約時の返金ポリシー | 必須 | 「途中解約しても日割り返金なし」等の方針を terms に明記 |

## 5. 将来の GDPR/CCPA 対応 (海外展開時)

| 項目 | 必要性 (海外展開時) | 対応案 |
|---|---|---|
| Cookie consent UI | 必須 (EU/UK 訪問時) | 第三者 consent management platform (CMP) 導入。例: Cookiebot / Osano |
| 「広告 cookie の opt-out」UI | 必須 | CMP 経由で個別 vendor の opt-out 切り替え |
| データポータビリティ請求対応 | 必須 (EU 居住者) | CSV / JSON エクスポート機能。アカウント設定画面に追加 |
| 「ユーザーデータの即時削除」期限 | 必須 (30 日以内推奨) | 既存「アカウント削除」機能で対応 ✅ (関連テーブル一括削除済) |
| 英語版法務ページ | 必須 | `/en/terms` `/en/privacy` `/en/contact` を新規作成 |
| 「同意の撤回」UI | 必須 (GDPR Art.7(3)) | CMP 経由で「同意の撤回」リンクを footer に常設 |

## 6. Plan B B-6 の結論

- **一般公開 (無料公開) は現状の法務文言と Plan B B-3 / B-4 の整備で対応可能**。追加実装は不要。
- **AdSense 申請時**は privacy §4 / §6 への追記 + ads.txt 追加が最小タスク。
- **サブスク導入時**は特商法ページ追加 + terms 有料プラン条項追加 + privacy 決済情報追記 + Stripe を外部サービスに追加が必須。
- **海外展開時**は CMP 導入 + 英語版ページ + データポータビリティ機能が必須。

これらは Phase 3 法務 / 収益化 plan で別途扱う。Plan B-6 では本レポートが成果物。
